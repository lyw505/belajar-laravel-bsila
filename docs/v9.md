# Scenario Laravel v9

## Skenario

* Skenario kali ini membuat backend jadi lebih **terstruktur**.
* Dengan cara **membagi beban controller** kepada fungsi yang sesungguhnya.
* Pertama, **Request** digunakan untuk memvalidasi dan mengelola input dari user (HTTP Request) sebelum diproses lebih lanjut.
* Kedua, **Service** digunakan untuk menyimpan logika bisnis dari sistem.
* Ketiga, **Repository** digunakan untuk mengelola akses ke database, seperti mengambil data atau menyimpan data.

Jadi alurnya ketika sebuah function di controller berjalan:

```
Request → Service → Repository
```

Sebagai contoh, skenario ini dapat diterapkan ketika akan menambah **siswa**.

---

## Langkah-langkah

### 1. Membuat class request untuk menambahkan siswa

Jalankan perintah:

```bash
php artisan make:request StoreSiswaRequest
```

Lalu modifikasi bagian berikut:

#### a. Ubah authorize:

**Sebelum:**

```php
public function authorize(): bool
{
    return false;
}
```

**Menjadi:**

```php
public function authorize(): bool
{
    return true;
}
```

#### b. Ubah rules:

**Sebelum:**

```php
public function rules(): array
{
    return [
        //
    ];
}
```

**Menjadi:**

```php
public function rules(): array
{
    return [
        'nama' => 'required|string|max:100',
        'tb'   => 'required|numeric|min:30|max:250',
        'bb'   => 'required|numeric|min:10|max:200',
    ];
}
```

---

### 2. Membuat Repository

Buat direktori baru bernama **Repositories** di dalam folder `app`.

Buat file `SiswaRepository.php` dan isi dengan kode berikut:

```php
<?php

namespace App\Repositories;

use App\Models\Siswa;

class SiswaRepository
{

}
```

Tambahkan function berikut di dalamnya:

```php
public function create(array $data)
{
    $admin = \App\Models\Admin::create([
        'username' => $data['nama'],
        'password' => bcrypt($data['nama']),
        'role'     => 'siswa',
    ]);

    $siswa = \App\Models\Siswa::create([
        'id'   => $admin->id,
        'nama' => $data['nama'],
        'tb'   => $data['tb'],
        'bb'   => $data['bb'],
    ]);

    return $siswa;
}
```

---

### 3. Membuat Service

Buat direktori baru bernama **Services** di dalam folder `app`.

Buat file `SiswaService.php` dan isi dengan kode berikut:

```php
<?php

namespace App\Services;

use App\Repositories\SiswaRepository;

class SiswaService
{

}
```

Tambahkan kode berikut:

```php
protected $repo;

public function __construct(SiswaRepository $repo)
{
    $this->repo = $repo;
}
```

Lalu tambahkan function berikut:

```php
public function createSiswa(array $data)
{
    return $this->repo->create($data);
}
```

---

### 4. Modifikasi Controller

Tambahkan kode berikut di dalam `SiswaController.php`:

```php
protected $service;

public function __construct(SiswaService $service)
{
    $this->service = $service;
}
```

Kemudian modifikasi function `store` menjadi:

```php
public function store(StoreSiswaRequest $request)
{
    $this->service->createSiswa($request->validated());
    return redirect()->route('home')->with('success', 'Data siswa berhasil ditambahkan!');
}
```

---

## Penjelasan

### Alur Umum

```
User → Route → Controller → Request → Service → Repository → Model (Database)
```

* User memasukkan field pada form input, lalu menekan tombol **Simpan**.
* Tombol simpan menjalankan route POST sesuai dengan rute yang tertulis, lalu menjalankan controller (`SiswaController@store`).

---

### Saat di Controller

```php
public function store(StoreSiswaRequest $request)
{
    $this->service->createSiswa($request->validated());
    return redirect()->route('home')->with('success', 'Data siswa berhasil ditambahkan!');
}
```

Artinya adalah menjalankan **request** dulu yang ada pada `StoreSiswaRequest`.

---

### Lalu Menjalankan Service pada Function createSiswa

```php
public function createSiswa(array $data)
{
    return $this->repo->create($data);
}
```

---

### Kemudian Menjalankan Repository pada Function create

Fungsi ini berisi logika untuk **menyimpan data** (berisi sesuai dengan field form yang telah divalidasi oleh class request) melalui **model `Siswa`** yang terhubung dengan tabel **datasiswa**.

---

## Tugas

Silakan merubah semua controller yang memang bisa dibagi bebannya dengan **Request**, **Service**, dan **Repository** sesuai dengan kebutuhan.

> **Note:**
> Tidak semua controller membutuhkan itu semua.
> Terkadang hanya membutuhkan salah satu atau dua saja.
> Bahkan ada kalanya tidak membutuhkan sama sekali.
